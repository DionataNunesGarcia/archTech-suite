# User Acceptance Testing (UAT) Guide

## Overview

UAT validates that each feature meets stakeholder expectations before go-live.

## Session Structure

### Pre-Session
1. Define test scenarios per squad
2. Prepare demo environment
3. Share feedback form with stakeholders

### During Session
1. Walk through each scenario
2. Stakeholder observes and interacts
3. Record feedback in structured form
4. Note severity: Critical / Major / Minor / Suggestion

### Post-Session
1. Triage feedback by severity
2. Assign action items
3. Schedule retest for critical/major items

## Scenarios per Squad

| Squad | Critical Flows | Scenarios |
|-------|---------------|-----------|
| CRM | Lead scoring, follow-up schedule | 3 scenarios |
| Proposals | Generation, contract review | 3 scenarios |
| Financeiro | Cashflow forecast, collection | 3 scenarios |
| Library | Document search, classification | 3 scenarios |
| Permits | Advisor, document generation | 3 scenarios |
| Facilities | Maintenance, warranty | 3 scenarios |

## Approval Criteria

- ≥ 80% of scenarios approved by stakeholders
- Zero critical or major unresolved issues
- All blockers documented with mitigation plan
